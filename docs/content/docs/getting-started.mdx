# Getting Started

## Installation

### Via Composer (Recommended)

```bash
composer require queuety/queuety
```

### Manual Installation

Download the latest release from GitHub, upload the zip to your WordPress plugins directory, and activate:

```bash
wp plugin install queuety.zip --activate
```

On activation, Queuety creates four database tables (`queuety_jobs`, `queuety_workflows`, `queuety_logs`, `queuety_schedules`) and registers its WP-CLI commands.

## Configuration

Queuety works out of the box with sensible defaults. All constants are optional. Define them in `wp-config.php` before the plugin loads:

| Constant | Default | Description |
|---|---|---|
| `QUEUETY_TABLE_JOBS` | `queuety_jobs` | Jobs table name |
| `QUEUETY_TABLE_WORKFLOWS` | `queuety_workflows` | Workflows table name |
| `QUEUETY_TABLE_LOGS` | `queuety_logs` | Logs table name |
| `QUEUETY_TABLE_SCHEDULES` | `queuety_schedules` | Schedules table name |
| `QUEUETY_TABLE_QUEUE_STATES` | `queuety_queue_states` | Queue states table name |
| `QUEUETY_TABLE_WEBHOOKS` | `queuety_webhooks` | Webhooks table name |
| `QUEUETY_RETENTION_DAYS` | `7` | Auto-purge completed jobs/workflows after N days |
| `QUEUETY_LOG_RETENTION_DAYS` | `0` | Auto-purge logs after N days (0 = keep forever) |
| `QUEUETY_MAX_EXECUTION_TIME` | `300` | Max seconds per job/step before timeout |
| `QUEUETY_WORKER_SLEEP` | `1` | Seconds to sleep when queue is empty |
| `QUEUETY_WORKER_MAX_JOBS` | `1000` | Max jobs before worker restarts (memory safety) |
| `QUEUETY_WORKER_MAX_MEMORY` | `128` | Max MB before worker restarts |
| `QUEUETY_RETRY_BACKOFF` | `exponential` | Backoff strategy: `fixed`, `linear`, or `exponential` |
| `QUEUETY_STALE_TIMEOUT` | `600` | Seconds before a processing job is considered stale |
| `QUEUETY_DEBUG` | `false` | Enable verbose worker logging |

## Dispatch your first job

Create a handler class that implements `Queuety\Handler`:

```php
use Queuety\Handler;

class SendEmailHandler implements Handler {
    public function handle( array $payload ): void {
        wp_mail( $payload['to'], $payload['subject'], $payload['body'] );
    }

    public function config(): array {
        return [
            'queue'           => 'emails',
            'max_attempts'    => 5,
            'needs_wordpress' => true,
        ];
    }
}
```

Register the handler and dispatch a job:

```php
use Queuety\Queuety;
use Queuety\Enums\Priority;

// Register the handler
Queuety::register( 'send_email', SendEmailHandler::class );

// Dispatch
Queuety::dispatch( 'send_email', [
    'to'      => 'user@example.com',
    'subject' => 'Welcome',
    'body'    => 'Thanks for signing up.',
] )->with_priority( Priority::High );
```

## Create your first workflow

Workflows break long-running processes into durable steps. Each step implements `Queuety\Step` and returns data that is merged into the workflow state:

```php
use Queuety\Step;

class FetchDataHandler implements Step {
    public function handle( array $state ): array {
        // $state contains the initial payload and all prior step outputs
        return ['user_name' => 'Alice', 'order_count' => 12];
    }

    public function config(): array {
        return ['needs_wordpress' => true];
    }
}

class GenerateSummaryHandler implements Step {
    public function handle( array $state ): array {
        // $state now includes user_name and order_count from the previous step
        $summary = "Report for {$state['user_name']}: {$state['order_count']} orders";
        return ['summary' => $summary];
    }

    public function config(): array {
        return [];
    }
}
```

Define and dispatch the workflow:

```php
$workflow_id = Queuety::workflow('weekly_report')
    ->then(FetchDataHandler::class)
    ->then(GenerateSummaryHandler::class)
    ->on_queue('reports')
    ->dispatch(['user_id' => 42]);
```

## Start a worker

Workers are long-running PHP processes started via WP-CLI:

```bash
# Start a worker on the default queue
wp queuety work

# Start a worker on a specific queue
wp queuety work --queue=emails

# Process one batch and exit (for cron)
wp queuety work --once

# Process all pending jobs and exit
wp queuety flush
```

Verify everything is working:

```bash
wp queuety status
```
